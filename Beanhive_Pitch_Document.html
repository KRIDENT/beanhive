Project Overview

Beanhive is a fully functional coffee shop e-commerce platform built from scratch. It replicates the core experience of major coffee chains — browsing a menu, ordering drinks, earning rewards, managing your account, and administering the business — all in a lightweight, framework-free package.

Customer Features

Authentication

  • Email/password registration and login with server-side validation
  • Google OAuth 2.0 one-click sign-in
  • Token-based password reset flow with email delivery
  • Email verification with resend capability
  • Persistent sessions (7-day secure httpOnly cookies)

Shopping Cart

  • Persistent cart via localStorage (survives page refresh and navigation)
  • Slide-in cart drawer with item list, quantities, and size display
  • Real-time subtotal, tax (8.25%), and total calculation
  • One-click checkout that creates a server-side order
  • Toast notification with item image on add-to-cart

Menu & Ordering

  • Browse by category with sidebar/tab filtering
  • Drink detail pages with description, nutrition, and size options
  • Order history with status tracking (Pending → Confirmed → Cancelled)
  • Reorder from previous orders

Rewards Program

  • Automatic star earning: 2 stars per $1 spent
  • Three-tier system: Green (0–649), Gold (650–1,199), Reserve (1,200+)
  • Visual milestone tracker with benefits at each level
  • Transaction history showing every earn/redeem event
  • Auto-enrollment on account creation

Favorites

  • Heart toggle on any drink card or detail page
  • Server-synced favorites list (persists across devices)
  • Dedicated Favorites tab in menu with quick reorder
  • Optimistic UI updates with instant visual feedback

Search

  • Global search modal triggered by Ctrl+K / Cmd+K
  • Searches menu items and site pages simultaneously
  • Results grouped by category with keyboard navigation
  • Cached menu data (5-minute TTL) for fast results

Dark Mode

  • Toggle in navbar with smooth sliding pill animation
  • Preference persisted in localStorage
  • Applied before render (no flash of wrong theme)
  • Full coverage across all 13 pages

Admin Features

User Management

  • View all registered users with role, email, and join date
  • Promote customers to admin or demote admins to customer
  • Role changes take effect immediately (checked on every API request)
  • Self-demotion protection (admins cannot remove their own role)

Menu Management (Full CRUD)

  • Create new menu items with name, category, price, calories, and description
  • Upload product images (PNG, JPG, WebP — 5MB limit)
  • Mark items as food vs. drink, and featured vs. standard
  • Edit all fields on existing items including image replacement
  • Delete items with confirmation
  • Filter menu items by category

Order Overview

  • View all orders across all users (last 50)
  • See customer name, item count, total, status, and date
  • Status badges for quick visual scanning

Analytics Dashboard

  • Summary cards: Total users, total orders, total revenue, today/week/month/all-time breakdowns
  • Revenue Over Time: Bar + line chart with 7/30/90/365-day selector
  • Orders by Status: Doughnut chart (pending/confirmed/cancelled)
  • Top 10 Selling Items: Horizontal bar chart by units and revenue
  • New Users Over Time: Line chart showing growth

Backend Architecture

The server follows a service-oriented architecture with 7 distinct services, each owning its own routes, controllers, services, and repositories. All services share a common database, middleware stack, and utility layer.

ServicePrefixResponsibility
Auth/v1/authGoogle OAuth 2.0 flow, session creation
User/v1/usersRegistration, login, logout, profile, password reset, email verification
Menu/v1/menuProduct catalog, categories, featured items, slug-based lookups
Orders/v1/ordersCheckout, order history, cancellation, tax calculation
Rewards/v1/rewardsStar balance, tier management, transaction history
Favorites/v1/favoritesAdd/remove/check favorites, synced to user account
Admin/v1/adminUser roles, menu CRUD, image upload, analytics

Database Schema

SQLite database with WAL mode enabled, foreign key constraints, and 9 tables:

TablePurpose
usersAccounts (name, email, hashed password, role, Google ID, verification status)
sessionsServer-side session storage
menu_itemsProduct catalog (slug, name, category, price, calories, image, featured flag)
ordersOrder records (items JSON, subtotal, tax, total, status)
favoritesUser-drink favorite pairings (unique constraint)
rewardsUser star balance and tier (one record per user)
rewards_transactionsStar earn/redeem history with order linkage
password_reset_tokensOne-time tokens with expiration and used flag
email_verification_tokensEmail verification tokens with expiration

Middleware Stack

  • Correlation ID — every request gets a unique trace UUID
  • Compression — gzip for all responses
  • CORS — origin whitelist (localhost + deployed URL)
  • Body Parsing — JSON and URL-encoded form data
  • Session — secure cookie-based sessions
  • Request Logger — logs method, path, status, and duration
  • Error Handler — standardized error responses with correlation ID

What Makes Beanhive Notable

Zero Framework Dependencies

No React, Vue, Angular, or Tailwind. Every component — the cart drawer, search modal, toast system, skeleton loaders, theme toggle — is built from scratch with vanilla HTML, CSS, and JavaScript. This demonstrates deep understanding of web fundamentals and produces a lightweight, fast-loading application.

Minimal Backend Footprint

The entire server runs on just 11 npm packages. Every dependency was chosen intentionally — bcrypt for security, multer for uploads, express-session for auth. No ORM, no bloated middleware chains, no unnecessary abstractions.

Complete User Journey

From landing page to checkout to rewards — every step of the customer experience is implemented. Users can browse, search, favorite, order, earn stars, track their tier, manage their profile, and recover their password. Admins can manage everything from a dedicated dashboard.

Production-Ready Architecture

Health check endpoints, structured error handling, gzip compression, request correlation IDs, environment-based configuration, secure cookie settings, and CORS policies — the application is built with deployment and observability in mind from day one.
